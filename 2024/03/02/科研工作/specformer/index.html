环境搭建

  • pip install -i https://pypi.tuna.tsinghua.edu.cn/simple -r requirements.txt，官方环境并不完整，还需要安装以下依赖

    • pip install -i https://pypi.tuna.tsinghua.edu.cn/simple rdkit
    • pip install torch_cluster-1.6.0+pt113cu117-cp310-cp310-linux_x86_64.whl
    • pip install torch_sparse-0.6.17+pt113cu117-cp310-cp310-linux_x86_64.whl
    • pip install torch_scatter-2.1.0+pt113cu117-cp310-cp310-linux_x86_64.whl
    • 以上依赖地址data.dgl.ai/wheels/repo.html

  • 之后会遇到dgl库版本不匹配的问题

    • 因为requirements.txt文件中的dgl库是cpu版本，需要torch1.13+cuda117版本的dgl库
    • 安装前要先删除原来的dgl库 pip uninstall dgl
    • pip install dgl-cu113 -f https://data.dgl.ai/wheels/repo.html

  • 之后又会报OSError: libcusparse.so.11: cannot open shared object file: No such file or directory

    • 参考以下文档解决成功解决 OSError: libcusparse.so.11: cannot open shared object file: No such file or directory-腾讯云开发者社区-腾讯云 (tencent.com)

    • 手动下载了libcusparse.so.11，并放置到python的site库中即可

    • 此时并未成功解决，还存在动态链接库路径问题，确保系统能够找到 libcusparse.so.11。

      • 使用以下命令检查动态链接库的路径:
      sudo ldconfig -p | grep libcusparse.so.11
      • 如果没有输出，表示系统找不到该库文件。你可以尝试将库文件路径添加到动态链接库搜索路径中：
      sudo echo "/usr/local/lib/python3.10/dist-packages/" >> /etc/ld.so.conf.d/local.conf
sudo ldconfig
      • 再使用以下命令检查动态链接库的路径：
      sudo ldconfig -p | grep libcusparse.so.11

    • 此时已经解决OSError问题

  • 成功运行前的最后一个问题可能会报没有checkpoint目录，直接在运行目录当级创建一个checkpoint目录
