Redis入门

概述

what？

Remote Dictionary Server

开源、使用C编写、支持网络、可基于内存亦可持久化的日志型、key-value数据库，并提供多种语言的api

Redis会周期性的把更新的数据写入磁盘或者将修改操作写入追加的记录文件。并在此基础上实现master-slave同步

结构化数据库

do what?

1.内存存储，持久化。（内存断电即失，so持久化很重要）

2.效率高，可用于高速缓存

3.发布订阅系统。

4.地图信息分析

5.计时器计数器（浏览量） ……

特性：

  1. 多样数据类型。
  2. 持久化。3.集群 4.事务 。。。。

事务

与MySQL的事务有所不同

一次性、顺序性、排他性：执行一些列的命令。

Redis事务本质：1.一组命令的集合。一个事务中的所有命令都会被序列化，在事务执行过程中，会按照顺序执行。

2.Redis事务没有隔离级别的概念。

所有的命令在事务中，并没有直接被执行。只有发起执行命令时才会执行。Exec()

特性3：不保证原子性

Redis单条命令保持原子性，但是事务不保证原子性。即三条命令也许有一条是错误的，但是并不妨碍另外两条的成功执行。

redis的事务：

  • 开启事务（） multi
  • 命令入队（） whatever
  • 执行事务（） exec

正常执行事务：

#启动
multi
OK
# 命令入队

127.0.0.1:6379> exec

秒杀案例——事务的延伸

秒杀：商品减去对应数量

秒杀成功者列入清单

核心逻辑

//秒杀过程
public static boolean doSecKill(String uid, String prodid) {
   //1. uid和prodid非空判断
   if (uid == null || prodid == null) {
      return false;
   }

   //2.连接Redis
   Jedis jedis = new Jedis("8.140.23.86", 6379);
   //3.拼接key
   //3.1 库存Key
   String storageKey = "sk:" + prodid + ":qt";
   //3.2秒杀成功用户key
   String userKey = "sk:"+prodid+":user";
   //4.获取库存，为空则说明秒杀未开始，等待
   String storage = jedis.get(storageKey);
   if (storage == null) {
      System.out.println("秒杀没有开始");
      jedis.close();
      return false;
   }
   //5. 判断用户是否重复秒杀操作
   if (jedis.sismember(userKey, uid)) {
      System.out.println("已经秒杀成功,不能重复秒杀");
      jedis.close();
      return false;
   }
   //6. 判断如果商品数量，库存数量<1，秒杀结束
   if(Integer.parseInt(storage) <= 0) {
      System.out.println("秒杀一结束");
      jedis.close();
      return false;
   }

   /*
   7. 秒杀过程
   库存-1  秒杀成功用户添加到清单中
    */
   jedis.decr(storageKey);
   jedis.sadd(userKey, uid);
   System.out.println("秒杀成功");
   jedis.close();
   return true;

}

Springboot整合

在Springboot2.x后原本使用的jedis被替换为lettuce

jedis:采用的直连，多个线程操作不安全。如果想要避免不安全，使用jedis pool连接池。更像BIO模式。

lettuce:采用netty,实例可以在多个线程中进行共享，不存在线程不安全的情况。可以减少线程数据，更像NIO模式。

自定义RedisTemplate

@Configuration
public class RedisConfig {
    //自定义RedisTemplate，一般企业中都有模板可以直接复制粘贴使用.
    @Bean
    @SuppressWarnings("all")
    public RedisTemplate<String,Object> redisTemplate(RedisConnectionFactory factory) {
        //字符串常用开发方便
        RedisTemplate<String, Object> template = new RedisTemplate<>();
        template.setConnectionFactory(factory);
        //json序列化配置
        Jackson2JsonRedisSerializer jackson2JsonRedisSerializer = new Jackson2JsonRedisSerializer(Object.class);
        ObjectMapper om = new ObjectMapper();
        om.setVisibility(PropertyAccessor.ALL, JsonAutoDetect.Visibility.ANY);
        om.enableDefaultTyping(ObjectMapper.DefaultTyping.NON_FINAL);
        jackson2JsonRedisSerializer.setObjectMapper(om);
        //String的序列化
        StringRedisSerializer stringRedisSerializer = new StringRedisSerializer();
        //key采用String的序列化方式
        template.setKeySerializer(stringRedisSerializer);
        //hash的key采用String的序列化方式
        template.setHashKeySerializer(stringRedisSerializer);
        //value的序列化采用jackson
        template.setValueSerializer(jackson2JsonRedisSerializer);
        //hash的value序列化采用Jackson
        template.setHashValueSerializer(jackson2JsonRedisSerializer);
        template.afterPropertiesSet();
        return template;
    }

}
